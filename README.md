# multi-label классификация для 41 финансового продукта клиентов банка
 
# Описание проекта
Цель проекта — разработать модель, предсказывающую вероятность открытия каждого из 41 банковского продукта (счета, карты, услуги) для заданного клиента. 
В отличие от стандартных рекомендательных систем, которые выдают топ-N продуктов, бизнесу требуются именно количественные оценки вероятности.

Банк располагает данными о 1 млн клиентов, из которых 750 тыс. используются для обучения, а 250 тыс. — для тестирования. 
Каждый клиент может владеть любым подмножеством из 41 банковского продукта. 
Данные полностью анонимизированы и обфусцированы: нет реальных названий продуктов и признаков.

Признаки:

Базовый набор — 200 признаков (категориальные cat_feature_i, числовые num_feature_j).

Дополнительный набор — более 2000 признаков (только числовые).

В признаках присутствуют пропуски и выбросы.

Целевая переменная — мультилейбл (41 бинарная метка, указывающая, открыт ли продукт у клиента).
Метрика соревнования — Macro Averaged ROC-AUC.

# Общий вывод

Проведен первичный анализ базового и дополнительного набора признаков.

На основе оценённой важности признаков были отобраны две подгруппы: 1300 признаков, суммарно дающих 90% важности, и 1600 признаков — 95% важности.»
Обучение проводилось на обеих подгруппах.

В решении использовались модели:

- CatBoostClassifier,
- multi‑label модель на основе HistGradientBoostingClassifier,
- multi‑label модель на основе XGBoost XGBClassifier,
- SketchBoost от pyboost.

Отдельно подбирались гиперпараметры модели HistGradientBoosting с помощью Optuna.

Лучшие значения макро‑усреднённой ROC‑AUC на тестовой выборке (250 тыс. клиентов) показали:

SketchBoost на 1300 признаках — 0.845678;

HistGradientBoosting (multi‑label) на 1600 признаках — 0.840862.

models/ — обученные модели. 

Эти папки находятся лишь формально: исходные данные и модели слишком велики для git.

notebooks/ preprocessing/ 

- kyberpolka-dataset-explore.ipynb: первичный анализ данных, построение графиков, первичный анализ важности признаков для дальнейшего формирования тренировочной выборки и обучения моделей

-  kyberpolka-xgb-features-1600.ipynb — обучение  multi‑label (XGBClassifierr) на наборе из 1600 наиболее важных признаков,

-  kyberpolka-pyboost.ipynb — обучение SketchBoost на наборе из 1300 наиболее важных признаков.
